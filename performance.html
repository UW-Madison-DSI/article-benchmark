Distribution observation

mg_gamma_product appears in 8 of 14 papers (57%). This is expected: it is the umbrella secondary model that combines gamma factors, so any paper using multiple environmental submodels includes it. Not a concern for extraction testing.

Structural variation by difficulty

Easy papers are neatly organized with clear headings. Adversarial papers bury the same information in dense paragraphs with no signposts. The pipeline should score higher on the easy ones.

Easy (3 papers)

  • 7 subsections with clear headers
  • Single equation, prominently displayed
  • No tables -- parameters inline in text
  • 1,087-1,160 words (shortest tier)

Medium (5 papers)

  • 8-10 subsections
  • 3-5 equations with labeled references
  • Tables present in all 5 papers
  • 1,240-1,357 words

Hard (4 papers)

  • 11-12 subsections
  • 6-7 equations -- highest density
  • Tables in 1 of 4 papers
  • 1,381-1,524 words (longest tier)

Adversarial (2 papers)

  • Only 2 subsections -- dense, flowing prose
  • 4 equations woven into paragraph text
  • No tables, no clear structural cues
  • Variable names used inconsistently
  • 1,293-1,301 words

Strong coverage overall

128 of 138 non-null parameter values (93%) appear verbatim in the generated text. Medium papers are nearly perfect (98%). This is critical -- if the extraction pipeline cannot find a value, it should be because the text is ambiguous, not because the value was never mentioned.

Gaps in easy papers

Synthetic002: Only 1 of 3 parameter values found in text (33%). The Gompertz model parameters (A, C) are described qualitatively rather than numerically. This is intentional for easy papers (fewer extraction cues), but worth tracking: the extraction pipeline should still find these values from contextual descriptions.

Synthetic009 and Synthetic011: 89% and 77% respectively. Both are hard papers with 6-7 equations -- some parameters are referenced by symbol only or described as "estimated from data" rather than given explicit numerical values.

Text quality indicators

The generated papers read like real methods sections. No obvious "written by AI" giveaways, proper citations, and correct domain terminology throughout.

Realism signals

  • Proper "Author et al." citation style throughout
  • 5-8 references per paper (realistic for methods sections)
  • Equation labels ([1], [2], ...) with in-text cross-references
  • Domain-appropriate language: cfu/mL, water activity, cardinal temperature
  • Methodology sections follow dairy science conventions
  • No first-person pronouns in easy/medium/hard papers

LLM artifact scan

Checked all 14 papers for common LLM tells: "delve", "as an AI", "it's important to note", "in conclusion", "I cannot". Zero detections except one minor instance of "it is worth noting" in Synthetic014 (adversarial). This is acceptable -- the phrase appears in real scientific papers.

Remaining concern

Adversarial papers use variable-name aliasing ("the test population", "the organism", "the bacterial count") to refer to the same quantity. This is intentional difficulty, but the extraction pipeline may struggle with disambiguation. This is the whole point of including adversarial papers.

Gaps and issues to address

The pilot is good enough to test the pipeline, but there are a few things to fix before we scale to 75 papers: more adversarial examples, cover the remaining templates, and add multi-section papers.

Before extraction pipeline run

  • Only 2 adversarial papers. Small sample may not be representative. Consider generating 2-3 more for statistical confidence.
  • 3 unused templates. mg_aw_salt, mg_log_linear_inactivation, mg_monte_carlo_threshold should be exercised in the full run.
  • Synthetic002 low parameter coverage (33%). Verify this is intentional difficulty, not a generation bug.
  • No multi-section papers. All papers are Methods-only. Real papers have Introduction, Results, Discussion.

Not blocking for pilot

  • Title similarity across papers (many share "Development of a quality assessment model for...")
  • Word count range is narrow (1,087-1,524). Real papers vary more.
  • All papers use "Author et al." placeholder citations
  • No supplementary materials or figure references

Acceptable for the pilot gate

These are known limitations of synthetic papers. The pilot tests whether the generation-extraction-scoring loop works at all, not whether synthetic papers are indistinguishable from real ones. Multi-section papers and broader variation will be addressed in Phase 3 (scale-up).

Next steps

  • Run extraction pipeline on all 14 pilot papers
  • Score extracted output against ground truth JSON files
  • Validate difficulty separation: easy > medium > hard > adversarial scores
  • Identify extraction failure modes -- which equation types or structures cause the most errors
  • Decision gate: if scores show meaningful difficulty separation, proceed to 75-paper generation across 6-8 domains
Phase 2 pilot gate target: difficulty rank correlation > 0.7
